# Тренажёр печати кода

Веб-приложение для тренировки набора кода: вы выбираете язык и тему, бесплатная модель через [OpenRouter](https://openrouter.ai/) генерирует короткий фрагмент (5–15 строк), вы набираете его вслепую под подсветкой. В реальном времени показываются WPM, точность, число ошибок и время.

## Схема интерфейса (ASCII)

```
+------------------------------------------------------------------+
|  Тренажёр печати кода                                             |
|  ИИ генерирует фрагмент — вы печатаете его с клавиатуры.          |
+------------------------------------------------------------------+
| [ Язык ▼ ]   [ Тема ▼ ]   [ Сгенерировать новый код ]             |
+------------------------------------------------------------------+
|  WPM | Точность | Ошибки | Время                                   |
+------------------------------------------------------------------+
|  ┌────────────────────────────────────────────────────────────┐  |
|  |  подсвеченный код: верно / ошибка / ещё не набрано + курсор │  |
|  │  (невидимое поле ввода поверх блока)                        │  |
|  └────────────────────────────────────────────────────────────┘  |
+------------------------------------------------------------------+

После завершения набора — модальное окно: итоги и кнопки «Повторить» / «Новый код».
```

## Стек

- React 18, TypeScript, Vite 5
- Tailwind CSS 3
- `prism-react-renderer` (`Highlight`, встроенный Prism с языками, тема `vsDark`) для подсветки по символам
- OpenRouter Chat Completions API, по умолчанию модель `qwen/qwen3.6-plus:free` (переопределяется `VITE_OPENROUTER_MODEL`)

Клиентское приложение без собственного бэкенда: запросы к OpenRouter идут из браузера (нужен ключ в `.env`).

## Установка и запуск

1. Установите [Node.js](https://nodejs.org/) (LTS).
2. Склонируйте или скопируйте проект в папку на диске.
3. В корне проекта выполните:

```bash
npm install
```

4. Создайте файл `.env` на основе примера:

```bash
copy .env.example .env
```

На macOS/Linux: `cp .env.example .env`

5. Вставьте ключ OpenRouter в `.env`.

6. Запуск dev-сервера (по умолчанию [http://localhost:5173](http://localhost:5173)):

```bash
npm run dev
```

7. Сборка для продакшена:

```bash
npm run build
```

Просмотр сборки: `npm run preview`

## Как получить API-ключ OpenRouter

1. Откройте [https://openrouter.ai/](https://openrouter.ai/) и войдите или зарегистрируйтесь.
2. Перейдите в раздел ключей API (Keys) в панели аккаунта.
3. Создайте новый ключ и скопируйте его.
4. Вставьте ключ в файл `.env` в переменную `VITE_OPENROUTER_API_KEY`.

Подробности и лимиты бесплатных моделей смотрите в документации OpenRouter.

## Переменные `.env`

| Переменная | Описание |
|------------|----------|
| `VITE_OPENROUTER_API_KEY` | Секретный ключ OpenRouter. Обязателен для генерации кода. Читается только через `import.meta.env` в Vite, в `localStorage` не сохраняется. |
| `VITE_OPENROUTER_MODEL` | Id модели OpenRouter (например `qwen/qwen3.6-plus:free`). Если не задан, используется `qwen/qwen3.6-plus:free`. |

Файл `.env` не коммитьте в репозиторий (он уже в `.gitignore`).

## Настройки тренажёра

| Настройка | Описание |
|-----------|----------|
| **Язык** | Язык генерируемого кода: JavaScript, TypeScript, Python, Go, Rust, C++. Влияет на промпт к модели и на грамматику подсветки. |
| **Тема** | Тематика фрагмента: алгоритмы, функции, классы, API-запросы, утилиты — задаёт акцент в пользовательском промпте. |
| **Сгенерировать новый код** | Новый запрос к OpenRouter; после ответа поле упражнения сбрасывается под новый текст. |
| **Повторить** (в модалке) | Тот же код, счётчики и таймер сбрасываются. |
| **Новый код** (в модалке) | Сброс и немедленная новая генерация с текущими языком и темой. |

Счётчики: **WPM** = \((\text{символы} / 5) / \text{минуты}\) (в процессе — по длине уже набранного текста, после завершения — по полной длине упражнения); **точность** — доля верных нажатий среди всех нажатий, добавивших символы; **ошибки** — число символов, не совпавших с ожидаемым на момент ввода; **время** — от первого нажатия клавиши в поле до завершения точного совпадения с целевым текстом.

**Tab** в поле ввода вставляет символ табуляции и не уводит фокус со страницы.

## Структура проекта

| Путь | Назначение |
|------|------------|
| `src/main.tsx` | Точка входа React, подключение глобальных стилей. |
| `src/App.tsx` | Сборка экрана: настройки, статистика, область набора, модалка результатов, вызов генерации. |
| `src/index.css` | Tailwind и анимация курсора набора. |
| `src/components/CodeTypingArea.tsx` | Подсветка по символам, цвета верно/ошибка/ожидание, оверлей-`textarea`, обработка Tab, скелетон при загрузке. |
| `src/components/StatsPanel.tsx` | Панель WPM, точности, ошибок, времени. |
| `src/components/SettingsPanel.tsx` | Выбор языка и темы, кнопка генерации. |
| `src/components/ResultsModal.tsx` | Итоговая статистика и действия после упражнения. |
| `src/hooks/useOpenRouter.ts` | POST на `https://openrouter.ai/api/v1/chat/completions`, разбор ответа и ошибок для UI. |
| `src/hooks/useTypingEngine.ts` | Состояние ввода, таймер, ошибки, завершение при полном совпадении. |
| `src/utils/statsCalculator.ts` | Расчёт WPM, точности и итогового снимка статистики. |
| `src/utils/prismLanguages.ts` | Маппинг языка приложения на id грамматики из `Prism.languages` пакета `prism-react-renderer`. |
| `src/types/index.ts` | Общие типы: язык, тема, снимки статистики. |
| `.env.example` | Шаблон переменных окружения. |
| `vite.config.ts` | Конфигурация Vite и плагина React. |
| `tailwind.config.js` | Конфигурация Tailwind (пути к файлам с классами). |
| `package.json` | Зависимости и скрипты `dev`, `build`, `preview`. |
